# DynamoDB Table Setup

Commands to create all tables used by the platform.
Run with your AWS CLI profile (replace `YOUR_AWS_PROFILE` with yours).

> Replace `YOUR_APP` with your application's short name (e.g. `acme`, `mybrand`).
> All table names in the codebase are configured via `NEXT_DYNAMODB_TABLE_*` env vars,
> so the actual names can be anything — just keep them consistent.

All tables use PAY_PER_REQUEST billing — no capacity planning needed.

---

## 1. `YOUR_APP-slots`

Stores available calendar slots and bookings.

- **PK:** `slotId` (String)
- **GSI:** `date-startTime-index` — used to query slots by date

```bash
aws dynamodb create-table \
  --table-name YOUR_APP-slots \
  --attribute-definitions \
    AttributeName=slotId,AttributeType=S \
    AttributeName=date,AttributeType=S \
    AttributeName=startTime,AttributeType=S \
  --key-schema \
    AttributeName=slotId,KeyType=HASH \
  --global-secondary-indexes '[
    {
      "IndexName": "date-startTime-index",
      "KeySchema": [
        {"AttributeName": "date", "KeyType": "HASH"},
        {"AttributeName": "startTime", "KeyType": "RANGE"}
      ],
      "Projection": {"ProjectionType": "ALL"}
    }
  ]' \
  --billing-mode PAY_PER_REQUEST \
  --region us-east-1 \
  --profile YOUR_AWS_PROFILE
```

## 3. `YOUR_APP-reviews`

Stores customer reviews with moderation status.

- **PK:** `reviewId` (String)
- **GSI:** `status-createdAt-index` — used to query reviews by status, newest first

```bash
aws dynamodb create-table \
  --table-name YOUR_APP-reviews \
  --attribute-definitions \
    AttributeName=reviewId,AttributeType=S \
    AttributeName=status,AttributeType=S \
    AttributeName=createdAt,AttributeType=S \
  --key-schema \
    AttributeName=reviewId,KeyType=HASH \
  --global-secondary-indexes '[
    {
      "IndexName": "status-createdAt-index",
      "KeySchema": [
        {"AttributeName": "status", "KeyType": "HASH"},
        {"AttributeName": "createdAt", "KeyType": "RANGE"}
      ],
      "Projection": {"ProjectionType": "ALL"}
    }
  ]' \
  --billing-mode PAY_PER_REQUEST \
  --region us-east-1 \
  --profile YOUR_AWS_PROFILE
```

## Verify all tables exist

```bash
aws dynamodb list-tables \
  --region us-east-1 \
  --profile YOUR_AWS_PROFILE \
  --query "TableNames[?starts_with(@, 'YOUR_APP')]" \
  --output table
```

Expected output:
```
YOUR_APP-contacts
YOUR_APP-landing-config
YOUR_APP-review-tokens
YOUR_APP-reviews
YOUR_APP-slots
```

## IAM permissions required

The runtime IAM user needs the following policy on all `YOUR_APP-*` tables.
Replace `YOUR_ACCOUNT_ID` with your 12-digit AWS account ID.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "dynamodb:GetItem",
        "dynamodb:PutItem",
        "dynamodb:UpdateItem",
        "dynamodb:DeleteItem",
        "dynamodb:Query",
        "dynamodb:Scan",
        "dynamodb:BatchWriteItem"
      ],
      "Resource": [
        "arn:aws:dynamodb:us-east-1:YOUR_ACCOUNT_ID:table/YOUR_APP-*",
        "arn:aws:dynamodb:us-east-1:YOUR_ACCOUNT_ID:table/YOUR_APP-*/index/*"
      ]
    }
  ]
}
```
